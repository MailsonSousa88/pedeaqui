# Onboarding Flow Pattern

Usado para fluxos multi-step do lojista.

## Estrutura

- Cabeçalho com logo e contexto.
- Indicador de etapa.
- Barra de progresso.
- Card principal com uma etapa por vez.
- Botões de navegação no rodapé.
- Revisão final com blocos editáveis.

## Regras

- Cada etapa deve ter um objetivo único.
- Botão principal avança ou finaliza.
- Botão voltar não deve perder dados preenchidos.
- Loading final deve impedir múltiplos envios.
