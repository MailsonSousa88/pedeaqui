# Frontend Specs

Specs executáveis do frontend ficam nesta pasta.

Cada feature deve seguir:

```text
specs/<id-feature>/
├── spec.md
├── clarify.md
├── plan.md
├── tasks.md
├── checklist.md
└── contracts/
```

Não criar specs placeholder incompletas. Criar a pasta da feature apenas quando a fase `specify` começar.
